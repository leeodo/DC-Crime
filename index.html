Data & Methods

Data Description

We used the following datasets to answer our questions: State Crime, DC crime dataset, demographic characteristics of DC by wards , economic characteristics of DC by wards, and housing characteristics of DC by wards. The description of the each data is as follows:


  • State Crime: Based on the Unified Crime Reporting Statistics and the collaboration of the U.S. Department of Justice and FBI, this dataset provides the crime rates and totals for states across the U.S. from 1960 to 2019. The crime reports are categorized into property (Burglary, Larceny, Motor) and violent crime (Assault, Murder, Rape, Robbery). We will use the subset containing only the information about the DMV area, partially as a reference for the other datasets.
  • DC Crime dataset is an incident-based dataset. Every row is an incident that contains detailed time elements, geographic elements, and crime category elements. This data is made public by the Washington, DC Metropolitan police department. The whole dataset includes data as early as eight years ago. So it’s a very comprehensive dataset.

  • Demographic characteristics of DC by wards contains variables age, sex, sace, ethnicity, total housing units, and voting age population by ward. This is a tidy data pulled from American Community Survey (ACS) 5-year data for 2020.

  • Economic characteristics of DC by wards contain variables to measure employment, commuting, occupation, income, health insurance, poverty, and more. This is a tidy data pulled from American Community Survey (ACS) 5-year data for 2020.

  • Housing characteristics of DC by wards contain variables on occupancy status, Units, Rooms, Year built, Owner/Renter (Tenure), Mortgage/Rent costs, and more. This is a tidy data pulled from American Community Survey (ACS) 5-year data for 2020.

Data Preprocessing

We preprocessed the data to make it easier to use. We used the following methods for each file:


DC Crime Dataset

  • After reading the data, the columns that are “datetime” type are converted to the corresponding type. Then data is aggregated by different types of tables for later use. This includes the number of crimes per day, the number of crimes per day by different types, the number of crimes per day per ward, and the number of violent crimes per ward in 2020, etc.

  • The latest data on social characteristics are in the year of 2020, thus above tables were later filtered for the year of 2020 only.

Demographic, Economic, Housing Characteristics of DC by wards

  • The three datasets are summarized in 8 wards and these data are later joined to existing datasets using the ward number.

DC Crime Map

This choropleth is meant to show the difference of overall crime situation across eight wards in Washington,D.C.

The data was first aggregated to the total number of crimes happened in 2020 per ward and then the crime rate was derived. Lastly, the aggregated data has been merged with the geojson file.

The color of each ward is based on the absolute value of crimes. A legend of colormap is provided for reference. This map has a hover feature. When users hover their mouse over the map, users can see specific details of crime statistics.

DC Violent Crime Map

This choropleth is meant to show the difference of violent crime situation across eight wards in Washington, D.C.

The data was first filtered by crime type to pick out violent crimes only then aggregated to the total number of crimes happened in 2020 per ward. Then the crime rate was derived. Lastly, the aggregated data has been merged with the geojson file.

The color of each ward is based on the absolute value of crimes. A legend of colormap is provided for reference. This map has a hover feature. When users hover their mouse over the map, users can see specific details of crime statistics.

DC Marker Clusters Map

This visualization is meant to show a more detailed distribution of crimes happened in DC. This used the same data as the overall crime choropleth since it contains longitude and latitude.

The markers are added one-by-one and the leaflet package is smart enough to aggregate markers based on location.

The map has many circles zooming in and out. The number on the circle represents the number of crimes happened in the area. When users hover their mouse on a circle, it will show a polygon corresponding to the area the specific circle represents. Furthermore, the color of circle is a gradient of number as green is low and orange is high.

DC Crime Heatmap

This heatmap is meant to show the density of crimes in Washington D.C., the folium heatmap function takes in location information like the marker cluster does. It automatically draws the density using color.

This map is continues in area and the density is represented from light orange(low) and red(high)
